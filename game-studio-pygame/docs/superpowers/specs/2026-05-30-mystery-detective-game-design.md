# 《暗室录》— 悬疑推理剧情游戏设计

## 概述

经典本格密室推理游戏，三案递进，先勘查后推理。纯文字AVG风格，800x600深色界面。

## 三案结构

| 案件            | 难度 | 时长    | 场景         | 嫌疑人 | 核心诡计 |
| --------------- | ---- | ------- | ------------ | ------ | -------- |
| 案一·书房密室   | 入门 | 5-6min  | 郊外别墅书房 | 3人    | 时间诡计 |
| 案二·暴风雪山庄 | 进阶 | 7-8min  | 山间旅馆     | 4人    | 身份替换 |
| 案三·钟楼终章   | 终局 | 8-10min | 古堡钟楼     | 5人    | 连环机关 |

三案共享隐藏线索网，案一细节为案三伏笔。

## 单案流程

```
序幕(案情介绍) → 勘查(探索场景/收集线索) → 审讯(证词/指认矛盾) → 推理(凶手/动机/手法) → 结案(评级/真相)
```

### 勘查阶段

- 文字描述场景 + 可调查物品列表
- 上下键选择、确认键调查
- 部分物品需特定顺序解锁隐藏线索

### 审讯阶段

- 逐个审讯嫌疑人，证词分段展示
- 玩家在证词中选择与物证矛盾之处
- 正确指认矛盾解锁额外线索

### 推理阶段

- 三个填空题：凶手？动机？手法？
- 每题3-5个选项

## 评分机制

每案满分100分：

- 勘查分（30分）：发现线索比例，隐藏线索额外加分
- 审讯分（30分）：正确指认矛盾数量
- 推理分（40分）：凶手(20) + 动机(10) + 手法(10)

S级需每案85+分。

## UI设计

全屏800x600，深色背景，四种界面状态：

- 场景界面：场景名称 + 描述文字 + 物品列表
- 审讯界面：嫌疑人简介 + 证词文本 + 矛盾指认区
- 推理界面：三个问题依次展示 + 选项按钮
- 结案界面：评级 + 得分明细 + 真相回顾

## 技术约束

- 必须使用SDK组件（Renderer, SceneManager, ScoreManager等）
- 禁止直接pygame调用、外部资源
- 所有配置通过config.yaml
- MVC结构：config.yaml → config.py → model.py → view.py → main.py
