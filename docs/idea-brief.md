# Terminal Game Studio — Idea Brief

## 一句话介绍

用自然语言描述游戏创意，Claude Code 自动生成、构建、运行一个可玩的 Terminal 小游戏。

## 核心定位

不是 "AI 写代码"，而是 **AI 驱动的 Terminal 游戏开发工作流**：

- Claude 负责思考（设计、调参、Debug）
- MCP 负责执行（建项目、编译、运行）
- Runtime 负责稳定（渲染、物理、输入）

## 目标用户

在 Claude Code / Cursor / Gemini CLI 等 AI Coding 工具中**等待代码生成**的开发者——将碎片等待时间转化为娱乐时间，无需切换窗口。

## 使用方式

```
用户: "做一个赛博朋克钓鱼游戏"
  ↓
Claude: 自动设计玩法 → 选择模板 → 生成配置 → 编译 → 运行
  ↓
用户: "敌人太快了"
  ↓
Claude: 自动调参 → 重新编译 → 运行
```

## 技术方案

| 层       | 技术选型                | 职责                               |
| -------- | ----------------------- | ---------------------------------- |
| AI 推理  | Claude Code + Skills    | 游戏设计、代码生成、自动 Debug     |
| 工具链   | MCP Server (TypeScript) | 项目管理、编译、运行、截图、调参   |
| 游戏引擎 | Go + BubbleTea SDK      | 渲染、物理、碰撞、场景、计分       |
| 游戏配置 | YAML DSL                | 游戏规则与参数，代码零修改即可调优 |

## 游戏模板（MVP）

- **Flappy** — Flappy Bird 类
- **Shooter** — 生存射击类
- **Rogue** — Roguelike 类

## 产品亮点

1. **自然语言开发**：说一句话就能生成一个可玩游戏
2. **自动修复循环**：编译失败 → Claude 自动读错修复 → 重新编译（最多 5 轮）
3. **AI 调参**：说 "太难了" 或 "太无聊" 自动调整游戏平衡
4. **60 秒规则**：所有游戏围绕 "一分钟" 主题设计，符合比赛要求
5. **Terminal 原生**：ASCII 风格、单手操作、即开即玩、"再来一局" 循环

## 比赛契合度

| 评分维度         | 契合点                                            |
| ---------------- | ------------------------------------------------- |
| "再来一局"成瘾性 | 即时重启 + 难度渐进 + 随机元素                    |
| 反常识设计       | 用户用自然语言创造意想不到的玩法组合              |
| AI 技术结合      | Claude Code 全链路 AI 驱动（设计→生成→调试→调优） |
| 一分钟主题       | 60 秒上限、碎片时间、快速上手                     |
